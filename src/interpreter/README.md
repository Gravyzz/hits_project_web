Проект состоит из следующих логических блоков:
Контекст выполнения (память программы)
Узлы программы (команды)
Узел выражений
Токены (элементы выражений)
Главный класс запуска.

Context — память программы
Класс Context отвечает за хранение состояния программы, а именно — всех переменных.
Он реализован через HashMap, где:
ключ — имя переменной
значение — её текущее числовое значение
Контекст позволяет:
объявлять переменные
изменять их значение
получать значение переменной при вычислении выражений
Таким образом, Context — это аналог оперативной памяти для интерпретируемой программы.

Token — элементы выражения
Класс Token используется для представления выражений в формализованном виде.
Каждое выражение разбивается на последовательность токенов:
CONST — числовая константа
VAR — переменная
OP — оператор
Например, выражение a + 5 представляется как набор токенов:
переменная a
константа 5
оператор +
Это упрощает дальнейшее вычисление выражений.

ExpressionNode — вычисление выражений
Класс ExpressionNode отвечает за вычисление арифметических выражений.
Он использует стек для вычислений и последовательно обрабатывает список токенов:
числа помещаются в стек
переменные заменяются их значениями из контекста
операторы берут два значения из стека и выполняют операцию
В результате работы возвращается одно целое число — результат выражения.

StatementNode — базовая инструкция
StatementNode — это интерфейс, который описывает любую исполняемую команду.
Каждая команда обязана реализовывать метод execute, который принимает Context и изменяет состояние программы.
Это позволяет добавлять новые типы команд без изменения существующего кода.
 
VarDeclNode — объявление переменных
Класс VarDeclNode реализует команду объявления переменной.
Он:
вычисляет выражение
сохраняет результат в контексте под указанным именем
Пример:
var a = 5;

AssignNode — присваивание
Класс AssignNode отвечает за изменение значения уже существующей переменной.
Он вычисляет выражение и обновляет значение переменной в контексте.
Пример:
a = a + 1;
ProgramNode — выполнение программы
ProgramNode представляет собой всю программу целиком.
Он содержит список инструкций и выполняет их последовательно, вызывая метод execute у каждой команды.
Таким образом, программа исполняется сверху вниз, как в обычных языках программирования.

Main — точка входа
Класс Main — это точка запуска приложения.
В нём:
создаётся контекст выполнения
вручную формируется программа из узлов
программа выполняется
результат выводится в консоль
Этот класс используется для тестирования и демонстрации работы интерпретатора.
